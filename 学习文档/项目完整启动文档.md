# OpenClaw 项目完整启动文档（源码版）

本指南用于把仓库代码从 0 启动到“可用状态”（Gateway 在线、可健康检查、可接入渠道/智能体）。

## 1. 成功标准

启动完成后，至少满足以下 3 条：

1. `pnpm openclaw gateway health` 输出 `OK`。
2. `pnpm openclaw channels status --probe` 输出 `Gateway reachable`。
3. 本机端口 `18789` 处于监听状态（Gateway 主端口）。

## 2. 首次准备（只需做一次）

### 2.1 环境要求

- Node.js `>=22.12.0`
- `pnpm`（仓库当前使用 `pnpm@10.23.0`）
- macOS / Linux / Windows(WSL2)

检查命令：

```bash
node -v
pnpm -v
```

### 2.2 安装依赖

在仓库根目录执行：

```bash
pnpm install
```

### 2.3 初始化 OpenClaw 本地配置与工作区

```bash
pnpm openclaw setup
```

这一步会初始化（或补齐）典型目录：

- `~/.openclaw/openclaw.json`
- `~/.openclaw/workspace`
- `~/.openclaw/credentials`

## 3. 启动方式（二选一）

不要同时使用两种方式，否则容易出现端口冲突和重复重启。

### 3.1 方式 A：服务模式（推荐长期运行）

适合日常常驻、重启后自动恢复。

1. 安装服务（首次）

```bash
pnpm openclaw gateway install --port 18789 --force
```

2. 启动服务

```bash
pnpm openclaw gateway start
```

3. 查看服务状态

```bash
pnpm openclaw gateway status --deep
```

### 3.2 方式 B：前台模式（推荐开发/调试）

适合你在当前终端直接看日志。

1. 若服务模式正在运行，先停掉：

```bash
pnpm openclaw gateway stop
```

2. 前台启动：

```bash
pnpm openclaw gateway run --bind loopback --port 18789 --force
```

说明：

- 命令会占用当前终端，按 `Ctrl + C` 停止。
- `--force` 会清理占用该端口的旧进程。

## 4. 启动后验证（必做）

```bash
pnpm openclaw gateway health
pnpm openclaw channels status --probe
```

检查端口（macOS / Linux）：

```bash
lsof -nP -iTCP:18789 -sTCP:LISTEN
```

打开控制台（Dashboard）：

```bash
pnpm openclaw dashboard
```

## 5. 关闭与重启

服务模式：

```bash
pnpm openclaw gateway stop
pnpm openclaw gateway restart
```

前台模式：

- 在运行终端按 `Ctrl + C`

## 6. 常用运行日志位置

- Gateway 服务日志：`~/.openclaw/logs/gateway.log`
- Gateway 服务错误日志：`~/.openclaw/logs/gateway.err.log`
- 运行期综合日志：`/tmp/openclaw/openclaw-YYYY-MM-DD.log`

常用查看命令：

```bash
tail -f ~/.openclaw/logs/gateway.log
tail -f ~/.openclaw/logs/gateway.err.log
```

## 7. 常见问题与处理

### 7.1 端口被占用（18789）

现象：启动时报 `Port 18789 is already in use`。

处理：

```bash
pnpm openclaw gateway stop
pnpm openclaw gateway run --bind loopback --port 18789 --force
```

如果你确认是其他程序占用：

```bash
lsof -nP -iTCP:18789 -sTCP:LISTEN
```

### 7.2 token 不匹配（Dashboard 无法连上）

现象：日志出现 `unauthorized: gateway token mismatch`。

处理：

```bash
pnpm openclaw dashboard
```

用该命令输出/打开的 Dashboard 地址进入，或在 UI 中更新正确 token。

### 7.3 `pnpm dev` 不是完整网关启动

`pnpm dev` 主要是 CLI 入口，不等于常驻 Gateway。

完整启动请使用：

- `pnpm openclaw gateway run ...`（前台）
- 或 `pnpm openclaw gateway start`（服务）

### 7.4 频道不工作

先看网关可达，再看频道状态：

```bash
pnpm openclaw gateway health
pnpm openclaw channels status --probe
```

如需首次接入渠道（例如 WhatsApp/Telegram/Discord），先执行：

```bash
pnpm openclaw channels login
```

## 8. 开发模式补充（与完整启动不同）

如果你只想做本地智能体开发并隔离状态：

```bash
pnpm gateway:dev
```

特点：

- 使用 dev profile（默认端口 `19001`）
- 默认跳过渠道启动（`OPENCLAW_SKIP_CHANNELS=1`）
- 适合调 Agent/工具，不适合验证真实渠道联通

## 9. 一键最小流程（可直接复制）

```bash
pnpm install
pnpm openclaw setup
pnpm openclaw gateway run --bind loopback --port 18789 --force
```

新终端验证：

```bash
pnpm openclaw gateway health
pnpm openclaw channels status --probe
```
